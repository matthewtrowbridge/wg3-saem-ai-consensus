# Source materials

Durable copies of the source documents that the WG3 working site draws on. Lives inside the repo so that a clone, fork, or future org migration carries the supporting material with it, not just the rendered HTML.

## What's here

### WG3 / AI education landscape
- **`AI_Competency_Frameworks_Brief.md`** — Matt Trowbridge's April 6, 2026 synthesis of published AI competency frameworks for medical education (Triola & Rodman 2025, Lee et al. 2024, AAMC CGEA, Tolentino et al. 2024, AI-PACE 2026, AIFM-ed 2025, AMEE Guide 178). Provides the landscape context for WG3's Aim 3 work and frames the "what's missing — WG3's unique contribution" positioning for the AEM E&T manuscript. Referenced from `resources.html` Literature landscape card.

### AHRQ R13 grant materials
Markdown extracts from the AHRQ R13 grant narrative authored by R. Andrew Taylor (UVA) and Jeremiah Hinson (Johns Hopkins). The same text is already publicly visible on the live site via the expandable `<details>` elements in the "AHRQ R13 foundation" card on `resources.html`, so committing the source files here does not disclose anything new — it just makes the supporting material durable inside the repo itself. Andrew has been notified separately; if he requests edits or removal, we will honor that immediately.

- **`AHRQ_R13_Specific_Aims.md`** — The three R13 aims (~470 words). Aim 3 is the WG3 mandate.
- **`AHRQ_R13_Project_Summary.md`** — The grant abstract (~350 words).
- **`AHRQ_R13_Conference_Plan.md`** — Sections A–E narrative plus the key-facts list (funding mechanism, co-chairs, attendees, keynote, original WG structure, past conference impact stats).
- **`Evidence_Brief_Process.md`** — How evidence briefs are built for each working group: AI-generated by the planning committee, librarian-reviewed, sourced from PubMed/Medline and Web of Science. Derived from the Slack canvas in `#resources-and-docs`.

**What is explicitly not included from the R13 grant, regardless of anything else:** budget, personnel cost detail, salary information, reviewer comments, internal critique, any pre-publication or draft sections.

## Copy policy

See the repo root `CLAUDE.md` for the three-tier rule that governs what goes in this folder. In short:

- **Safe to copy:** Matt's own writing, public links, anything Matt authored from scratch.
- **Ask the author first:** Andrew's grant materials, Michael's prior work, anything derived from someone else's draft or pre-publication content.
- **Link, don't host:** Third-party copyrighted papers.

Matt made a one-time judgment call on the AHRQ R13 materials above because the same text is already publicly visible via the site's inline expandable details — committing the source files here is a durability move, not a disclosure move. The rule as written still governs future additions and any new R13 content.
